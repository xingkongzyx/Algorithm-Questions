# 224 vs 227：栈的使用区别

## 题目差异

|  | 224. Basic Calculator | 227. Basic Calculator II |
|---|---|---|
| **运算符** | `+` `-` | `+` `-` `*` `/` |
| **括号** | ✅ 有 | ❌ 没有 |
| **优先级问题** | 无（加减同级） | 有（乘除优先于加减） |
| **示例** | `"1-(2+3)"` = -4 | `"3+2*2"` = 7 |

> **核心矛盾不同**：224 的难点是**括号嵌套**，227 的难点是**运算符优先级**。

---

## 为什么要使用栈？

### 224：为什么需要栈来处理括号？

**问题本质**：括号创造了"嵌套子表达式"，外层的计算必须**暂停**，等内层算完才能继续。

考虑表达式 `"1-(2+3)"`：
1. 我们从左到右扫描，先算出 `1`，遇到 `-`，准备减去后面的东西
2. 但后面不是一个简单的数字，而是 `(2+3)` —— 一个子表达式
3. 我们必须**暂时搁置**外层的 `res=1` 和 `sign=-1`，先进入括号内部计算
4. 括号内算出 `5` 后，再**回到外层**，执行 `1 + (-1) * 5 = -4`

**如果没有栈会怎样？** 进入括号时 `res` 被重置为 0 来计算内层，那外层的 `res=1` 和 `sign=-1` 就丢失了，出括号后无法恢复。

**栈的作用**：在进入括号前，把外层的"进度"（`res` 和 `sign`）保存到栈中；出括号时从栈中取回，继续外层的计算。这就像玩游戏时的**存档/读档**机制。

> 如果有多层嵌套括号（如 `"1-(2+(3-4))"`），每进入一层就存档一次，每退出一层就读档一次——这正是栈的 LIFO 特性完美匹配的场景。

### 227：为什么需要栈来处理优先级？

**问题本质**：乘除的优先级高于加减，但我们是从左到右扫描的，遇到加减时**不知道后面是否有乘除**。

考虑表达式 `"3+2*2"`：
1. 扫描到 `3`，遇到 `+`，能直接把 `3` 加到结果里吗？可以，因为 `3` 前面是 `+`
2. 扫描到 `2`，遇到 `*`，能直接把 `2` 加到结果里吗？**不行！** 因为 `2` 后面跟着 `*`，`2` 应该先和后面的数相乘
3. 但此时我们还没扫描到后面的数，所以 `2` 必须先**暂存**起来
4. 扫描到最后的 `2`，发现前面是 `*`，于是取出暂存的 `2`，计算 `2*2=4`，再存回去
5. 最后把栈里所有数加起来：`3 + 4 = 7`

**如果没有栈会怎样？** 遇到 `+` 时直接 `res += 2`（得到 `res=5`），然后遇到 `*2`，但 `2` 已经混入 `res` 中无法单独取出来做乘法了。

**栈的作用**：把每个数字按照"待加减"的形式暂存到栈中。遇到乘除时，从栈顶取出上一个数立即运算后放回；遇到加减时，直接把数字（正或负）压入栈。最后统一求和。栈充当了一个**延迟计算的缓冲区**，让乘除可以"插队"优先执行。

## 栈的用途对比

### 224 —— 栈用于保存括号外层的上下文

```python
# 遇到 '(' → 保存当前状态到栈，重置
stack.append(res)   # 保存外层已算出的结果
stack.append(sign)  # 保存括号前的运算符（+1 或 -1）
res = 0             # 重置，开始算括号内部
sign = 1

# 遇到 ')' → 从栈恢复外层状态，合并
res += num * sign        # 先把括号内最后一个数算完
num = 0
lastSign = stack.pop()   # 恢复括号前的运算符
leftRes = stack.pop()    # 恢复外层结果
res *= lastSign          # 用运算符作用于括号内结果
res += leftRes           # 与外层结果合并
```

- **栈中元素**：数字和运算符（+1/-1）交替存储 `[res, sign, res, sign, ...]`
- **何时操作栈**：仅在遇到 `(` 和 `)` 时
- **结果累积**：直接在 `res` 上即时累加（`res += num * sign`）
- **最终返回**：`res += num * sign; return res`

### 227 —— 栈用于暂存待加减的数字

```python
# 每遇到运算符或字符串末尾，根据 prevOp 决定数字如何入栈
if prevOp == "+":
    stack.append(curNum)              # 加法：正数入栈
elif prevOp == "-":
    stack.append(-curNum)             # 减法：负数入栈
elif prevOp == "*":
    stack.append(stack.pop() * curNum)   # 乘法：立即与栈顶运算
elif prevOp == "/":
    stack.append(int(stack.pop() / curNum))  # 除法：立即与栈顶运算

# 最终求和
res = sum(stack)
```

- **栈中元素**：只有数字（正数或负数）
- **何时操作栈**：每遇到一个运算符或字符串末尾都操作
- **结果累积**：数字先存入栈，最后统一求和
- **最终返回**：`return sum(stack)`

---

## 结果累积方式的根本区别

```
224：res 即时累加，栈仅在括号时使用
     res += num * sign  →  遇到 (  →  push(res, sign)  →  遇到 )  →  pop 合并

227：数字入栈，乘除立即运算，加减最后求和
     +/- → append(±num)  →  */÷ → pop*num再push  →  最后 sum(stack)
```

## 用例子直观理解

### 224：`"1-(2+3)"`

| 字符 | 动作 | res | num | sign | stack |
|:---:|---|:---:|:---:|:---:|:---:|
| `1` | 拼接数字 | 0 | 1 | 1 | `[]` |
| `-` | `res += 1*1 = 1`，sign=-1 | 1 | 0 | -1 | `[]` |
| `(` | `push(1, -1)`，重置 | 0 | 0 | 1 | `[1, -1]` |
| `2` | 拼接数字 | 0 | 2 | 1 | `[1, -1]` |
| `+` | `res += 2*1 = 2`，sign=1 | 2 | 0 | 1 | `[1, -1]` |
| `3` | 拼接数字 | 2 | 3 | 1 | `[1, -1]` |
| `)` | `res += 3*1 = 5`；pop → `5*(-1)+1 = -4` | -4 | 0 | 1 | `[]` |

→ 返回 **-4** ✅

### 227：`"3+2*2"`

| 字符 | 动作 | curNum | stack |
|:---:|---|:---:|:---:|
| `3` | 拼接数字 | 3 | `[]` |
| `+` | `prevOp="+"`, `append(3)` | 0 | `[3]` |
| `2` | 拼接数字 | 2 | `[3]` |
| `*` | `prevOp="+"`, `append(2)` | 0 | `[3, 2]` |
| `2` | 拼接数字（末尾触发） | 2 | `[3, 2]` |
| 末尾 | `prevOp="*"`, `pop()*2=4`, `append` | 0 | `[3, 4]` |

→ `sum([3, 4])` = **7** ✅

## 为什么不能互换方案？

### 224 不能用 227 的栈方案

227 的方案**无法处理括号**。例如 `"1-(2+3)"`：
- 遇到 `2` 时 `prevOp` 是 `-`，会把 `-2` 入栈
- 遇到 `3` 时 `prevOp` 是 `+`，会把 `3` 入栈
- `sum = 1 + (-2) + 3 = 2` ❌（正确答案是 `-4`）

括号改变了运算的分组方式，227 的"先存后加"策略无法感知这种分组。

### 227 不能用 224 的即时累加方案

224 的 `res += num * sign` **无法处理优先级**。例如 `"3+2*2"`：
- 遇到 `*` 时会先执行 `res += 2`（变成 5）
- 然后无法正确处理乘法，因为 `2` 已经被加到 `res` 里了
- 即时累加无法"撤回"已经加上去的数字来做乘除

---

## 总结

| 对比维度 | 224 | 227 |
|---|---|---|
| **栈的角色** | 括号上下文的"存档点" | 中间数字的"暂存区" |
| **栈中内容** | `(res, sign)` 成对存储 | 单个数字（正或负） |
| **操作频率** | 仅在 `(` `)` 时 | 每遇到运算符都操作 |
| **结果来源** | `res` 变量 | `sum(stack)` |
| **解决的核心问题** | 括号嵌套 | 运算符优先级 |

> 如果需要**同时处理括号和四则运算**（772. Basic Calculator III），则需要将两种策略结合：用**递归/栈保存括号上下文** + **227 的栈策略处理优先级**。
